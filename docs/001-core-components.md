# SQL 처리기 핵심 구성

## 1. REPL SQL 입력 처리

사용자가 터미널에서 SQL을 한 줄씩 입력하고, `exit` 또는 `quit`으로 종료할 수 있어야 한다.

## 2. SQL 파싱

`SELECT`, `INSERT` 문을 단순한 규칙으로 분석해 실행에 필요한 계획으로 바꿔야 한다.

## 3. 실행

파싱된 계획을 기준으로 `SELECT` 전체 조회 또는 `INSERT` 행 추가를 실행해야 한다.

## 4. 테이블 메타데이터 / 파일 저장

고정된 테이블 정보를 기준으로 테이블 이름을 컬럼 목록과 CSV 파일 경로에 연결하고, CSV 파일을 읽거나 써야 한다.

## 전체 흐름

```text
REPL SQL 입력
-> SQL 파싱
-> 실행 분기
-> SELECT 전체 조회 또는 INSERT 행 추가
-> CSV 읽기/쓰기
```

이 4가지가 현재 구현된 SQL 처리기를 구성하는 최소 단위다.
