# 수요 코딩회: B+ 트리 인덱스 구현

## 목적

- 단 하루, AI를 도구 삼아 팀별 프로젝트에 몰입한다.
- 결과물뿐만 아니라 과정도 중요하다. AI가 작성한 코드를 완벽히 내 것으로 소화해 그 원리를 설명할 수 있어야 한다.
- 낯선 기능도 Top-down으로 빠르게 풀어내며 학습 민첩성을 극대화하고, 현업 레벨의 기술을 직접 부딪쳐보며 실무 감각을 깨운다.

## 요구사항

이번 주 과제는 `B+ 트리 인덱스` 구현이다.

- 실제 DBMS에서 사용하는 디스크 기반 `B+ Tree`를 단순화한 메모리 기반 방식으로 구현한다.
- 테이블에 레코드가 추가되면 ID가 자동으로 부여되어야 한다.
- 자동 부여된 ID를 B+ 트리에 인덱스로 등록한다.
- ID 기반 검색 시 B+ 트리 인덱스를 활용할 수 있도록 만든다.
- 대규모 데이터 삽입 후 성능 테스트를 진행한다.
- 이전 차수에서 구현한 SQL 처리기를 그대로 사용하여 연동한다.
- 구현 언어는 C 언어를 사용한다.

대규모 데이터 성능 테스트 조건:

- 최소 1,000,000개 이상의 레코드를 INSERT한다.
- ID 기준 SELECT는 인덱스를 사용한다.
- 다른 필드 기준 SELECT는 선형 탐색을 사용한다.
- 두 SELECT 방식의 속도 차이를 비교한다.

구현의 우선순위:

- 학습보다 구현을 우선한다.
- AI 등을 적극적으로 활용해 결과물을 만들어내는 것을 최우선 목표로 한다.
- 다만, 만들어진 결과물 중 핵심 로직은 반드시 이해하고 설명할 수 있어야 한다.
- 코드를 직접 작성하기 어려운 경우 전체 코드를 AI를 통해 생성해도 괜찮다.
- 생성된 코드는 반드시 학습하고 원리를 이해해야 한다.

추가적으로, 다른 팀과 차별점을 둘 수 있는 구현 요소도 함께 고민한다.

## 중점 포인트

- `WHERE id = ?` 조건일 때 B+ 트리를 어떻게 사용할 것인지
- 대용량 레코드를 쉽게 생성하고 테스트할 수 있는 방법
- 이전 차수에서 만든 SQL 처리기와 매끄럽게 연결하는 방법

## 품질 기준

- 단위 테스트를 통해 함수를 검증한다.
- 기능 테스트를 통해 B+ 트리가 올바르게 적용되는지 확인한다.
- 엣지 케이스를 최대한 고려한다.
- 이력서와 포트폴리오에 포함할 수 있을 만큼 완성도 높은 수준으로 구현한다.
