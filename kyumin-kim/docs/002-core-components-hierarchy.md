# SQL 처리기 핵심 구성 계층화

## 1. REPL SQL 입력 처리

사용자가 터미널에서 SQL을 한 줄씩 입력하고, 입력된 SQL을 파싱과 실행 단계로 넘기는 영역이다.

### 1.1 프롬프트 출력

사용자가 SQL을 입력할 수 있도록 REPL 프롬프트를 출력한다.

예시:

```text
mini-db>
```

### 1.2 SQL 한 줄 읽기

표준 입력에서 SQL 문자열 한 줄을 읽고, 앞뒤 공백과 개행을 제거한다.

### 1.3 특수 명령 처리

특수 명령은 마침표(`.`)로 시작한다. 현재 지원되는 명령은 `.exit`이며, 입력값이 `.exit`이면 REPL을 종료한다.

## 2. SQL 파싱

입력된 SQL을 단순한 규칙으로 분석해 실행에 필요한 계획으로 바꾸는 영역이다. SQL은 반드시 세미콜론(`;`)으로 끝나야 한다.

### 2.1 SQL 타입 판별

SQL이 세미콜론(`;`)으로 끝나는지 먼저 확인한 뒤, 문장이 `SELECT`인지 `INSERT`인지 구분한다.

### 2.2 SELECT 테이블명 추출

`select * from 테이블;` 형태에서 `select`, `*`, `from`, 테이블 이름, `;`, 입력 끝을 순서대로 확인한다.

예시:

```sql
select * from users;
```

### 2.3 INSERT 테이블명과 값 목록 추출

`insert into 테이블 values (...);` 형태에서 `insert`, `into`, 테이블 이름, `values`, 값 목록, `;`, 입력 끝을 순서대로 확인한다.

예시:

```sql
insert into users values (1, kim);
```

## 3. 실행

파싱된 계획을 기준으로 `SELECT` 또는 `INSERT` 동작을 실행하는 영역이다.

### 3.1 실행 분기

파싱 결과의 타입을 보고 `SELECT` 실행 함수 또는 `INSERT` 실행 함수를 호출한다.

### 3.2 SELECT 전체 조회

테이블 CSV 파일을 읽고, 컬럼명과 모든 행을 출력한다.

### 3.3 INSERT 행 추가

파싱된 값 목록을 테이블 CSV 파일 끝에 한 줄로 추가한다.

## 4. 테이블 메타데이터 / 파일 저장

고정된 테이블 정보를 기준으로 테이블 이름, 컬럼 목록, CSV 파일 경로를 연결하고 파일을 읽거나 쓰는 영역이다.

### 4.1 테이블 이름 매핑

테이블 이름을 컬럼 목록과 CSV 파일 경로가 들어 있는 메타데이터로 변환한다.

예시:

```text
users -> data/users.csv
```

### 4.2 CSV 파일 열기

조회할 때는 읽기 모드로, 저장할 때는 추가 모드로 CSV 파일을 연다.

### 4.3 CSV 읽기/쓰기

`SELECT`에서는 CSV의 모든 행을 읽어 출력하고, `INSERT`에서는 값 목록을 CSV 한 줄로 쓴다.
