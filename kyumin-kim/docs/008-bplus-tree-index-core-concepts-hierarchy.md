# B+ 트리 인덱스 핵심 개념 계층화

## 1. 인덱스

인덱스는 특정 컬럼 값을 기준으로 레코드를 빠르게 찾기 위한 보조 구조다. 이 과제에서는 `id` 값을 기준으로 파일 안의 레코드 위치를 찾는 것이 핵심이다.

### 1.1 인덱스 key

인덱스 key는 검색 기준이 되는 값이다. 이 과제에서는 `WHERE id = ?` 조건의 `id` 값이 key가 된다.

### 1.2 인덱스 value

인덱스 value는 key로 찾은 결과가 실제 레코드와 이어지도록 하는 값이다. 파일 기반 DB에서는 레코드 자체가 아니라 파일 안의 레코드 위치가 value 역할을 한다.

### 1.3 인덱스 검색

인덱스 검색은 key를 이용해 필요한 레코드 위치를 찾는 과정이다. 검색이 성공하면 전체 파일을 순서대로 확인하지 않고 해당 레코드에 접근할 수 있다.

## 2. B+ 트리

B+ 트리는 많은 key를 정렬된 상태로 관리하면서 검색 경로를 짧게 유지하는 균형 트리다. 대량의 레코드가 추가되어도 `id` 검색이 전체 탐색보다 빠르게 동작해야 한다.

### 2.1 정렬된 key 관리

B+ 트리는 key를 정렬된 상태로 유지한다. key가 정렬되어 있어야 검색할 때 불필요한 범위를 빠르게 제외할 수 있다.

### 2.2 균형 유지

B+ 트리는 데이터가 많아져도 특정 방향으로만 길어지지 않아야 한다. 균형이 유지되어야 많은 레코드가 있어도 검색 경로가 짧게 유지된다.

### 2.3 leaf 중심 검색 결과

B+ 트리에서는 최종 검색 결과가 leaf 영역에서 결정된다. `id` 검색도 leaf에서 해당 key와 연결된 파일 내 레코드 위치를 찾는 흐름이 된다.

## 3. 파일 내 레코드 위치

파일 기반 DB에서는 레코드가 파일에 저장되므로, 인덱스 검색 결과는 실제 데이터 자체가 아니라 파일에서 레코드를 다시 찾기 위한 위치 정보여야 한다.

### 3.1 파일 저장 순서

레코드는 INSERT되는 흐름에 따라 파일에 저장된다. 같은 파일 안의 각 레코드는 서로 다른 위치를 가진다.

### 3.2 레코드 위치 정보

레코드 위치 정보는 파일 안에서 특정 레코드를 다시 찾기 위한 기준이다. 인덱스는 `id`와 이 위치 정보를 연결해야 한다.

### 3.3 위치 기반 읽기

위치 기반 읽기는 파일 전체를 탐색하지 않고 필요한 레코드가 있는 지점으로 이동해 데이터를 읽는 개념이다. 이 동작이 가능해야 인덱스 검색 결과가 실제 SELECT 결과로 이어진다.

## 전체 계층 구조

```text
B+ 트리 인덱스 핵심 개념
├── 1. 인덱스
│   ├── 1.1 인덱스 key
│   ├── 1.2 인덱스 value
│   └── 1.3 인덱스 검색
├── 2. B+ 트리
│   ├── 2.1 정렬된 key 관리
│   ├── 2.2 균형 유지
│   └── 2.3 leaf 중심 검색 결과
└── 3. 파일 내 레코드 위치
    ├── 3.1 파일 저장 순서
    ├── 3.2 레코드 위치 정보
    └── 3.3 위치 기반 읽기
```

## 검색 흐름

```text
id 값 확인
-> B+ 트리 인덱스에서 key 검색
-> leaf에서 파일 내 레코드 위치 확인
-> 해당 위치의 레코드 읽기
-> SELECT 결과 반환
```
